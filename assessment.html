Assessment & Readiness Framework

Clients may target any stage (even Stage 5), but they can only enter that stage if its entry criteria are met. This framework shows what is checked and what must already exist without prescribing delivery templates or methods.

Stage 0: Maturity Assessment & Target State

Determine where you are today, where you want to be, and what must be true to start there
0

What is assessed

Readiness is evaluated across data, metrics, planning, decision automation, and governance so the target stage is achievable and safe.

Data Architecture Standarized KPIs Planning Integration Decision Logic Governance & Risk Operating Model

Outputs (what will be produced)

  • Current maturity snapshot (by domain/function)
  • Target stage selection and rationale
  • Readiness against target stage entry criteria
  • Gap list + dependency risks
  • Minimum bridge needed to safely start the target stage
Key principle: “Target any stage” is allowed. “Start at any stage” depends on passing the stage gate.

Stage 1 gate

Outcome: Enterprise-grade Trusted
Data Foundation
1
Entry criteria
  • Priority sources identified and accessible
  • Target architecture agreed (lake/warehouse/lakehouse)
  • Data quality controls and stewardship defined
  • Security and access model established
  • Domain ownership and accountability set

Stage 2 gate

Outcome: Unified and Standardized
Performance Management
2
Entry criteria
  • Stage 1 trusted data foundation available
  • Standarized KPI library (definitions + formulas + grain)
  • Semantic layer / metrics governance in place
  • Reporting standards and lifecycle controls defined
  • Adoption model for self-service is governed

Stage 3 gate

Outcome: Forecast-driven
Planning Discipline
3
Entry criteria
  • Stage 2 Standarized KPIs and consistent history
  • Sufficient historical depth + stable drivers
  • Curated features and validation data available
  • Monitoring requirements defined (accuracy/drift)
  • Planning touchpoints identified for integration

Stage 4 gate

Outcome: Optimized Decisions
Across Priority Processes
4
Entry criteria
  • Decision catalog prioritized (where optimization helps)
  • Constraints & policy rules explicitly defined
  • Objective functions & trade-offs agreed
  • Workflow integration points are identified
  • Benefits measurement method is defined

Stage 5 gate

Outcome: Controlled Autonomy
With Embedded Oversight
5
Entry criteria
  • Stage 4 decision logic is proven and stable
  • Autonomy boundaries + escalation paths defined
  • Human-in-the-loop oversight model established
  • Auditability: logs, lineage, traceability requirements
  • Risk controls: drift, compliance, safety monitoring

Note: If a client “wants Stage 5,” Stage 0 validates Stage 5 entry criteria. If gaps exist, the output is the minimum bridge (often selective Stage 2 metric certification + Stage 4 decision policy) required to safely operate autonomy.
